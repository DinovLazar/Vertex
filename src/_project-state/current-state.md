## Division theming
- **Detection:** path-based via `getDivisionFromPath(pathname)` in `src/lib/divisions.ts` — `/consulting*`, `/marketing*`, else `shared`.
- **Provider:** `DivisionProvider` reads the pathname, emits `<div data-division={division}>`. No React context is used.
- **CSS:** `[data-division="consulting"]` and `[data-division="marketing"]` blocks in `globals.css` override both the shadcn HSL tokens (`--background`, `--foreground`, `--primary`, …) and the bespoke `--division-*` tokens. The default (shared) values sit on `:root`.
- **Components read `var(--division-accent)`, `var(--division-text-primary)`, etc.** via inline `style={{...}}` or bracket classes `text-[var(--division-text-secondary)]`. Tailwind utilities (`text-consulting-400`, `bg-marketing-bg`) still work and are independent of the data-division swap.

## Known issues / technical debt
- **BlogCard dates on `/mk` render in English format.** `BlogCard.tsx` uses next-intl's `useFormatter().dateTime()` — server and client agree (no hydration mismatch), but Node's Intl data for `mk-MK` on this Windows runtime falls back to English. Phase 15F addresses; working as-is for now.
- **`lucide-react@1.8.0` ships no brand icons.** This is a misleading version number — it's not the usual 0.x Lucide library. Footer needs inline SVGs for LinkedIn/Instagram/Facebook as a result. If a future phase upgrades the package, those inline SVGs can be swapped back to lucide imports.
- **`blog/[slug]/page.tsx` has no data loader or `generateStaticParams`.** Build of SSG output may behave unexpectedly; no reproduction steps yet.
- **`src/types/index.ts`** duplicates `NavItem` from `src/config/navigation.ts` with a weaker shape. Treat the config one as canonical.
- **Navbar mobile overlay covers `ScrollProgress`** — the bar is `z-60` but the overlay is `z-40`, so this is fine; noted here only so future refactors don't flip the order.
- **Dev server must be restarted after edits to `src/proxy.ts`.** Turbopack caches the compiled edge handler under `.next/dev/server/edge/` and can fail to pick up changes. `rm -rf .next` + preview restart is the fix.
